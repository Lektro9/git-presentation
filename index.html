GIT

präsentiert von Lars Kroll

Was ist Git?



Version-Control-System (VCS)
Erfinder: Linus Torvalds in 2005
"distributed" oder "decentralized"
(jeder kann lokal arbeiten)
Sammelt wichtige Informationen
"Wer hat welche Änderungen Wo gemacht?"
Rollback einfach gemacht
Möglichkeit jederzeit auf "ältere Version" zurück zu setzen

Warum Git/Github?

  • Teamwork
  • Cloudspeicherung (Github)
  • Feedback/starke Community
  • zum Angeben!
  • man brauch es sowieso irgendwann...
    1. erleichtertes Arbeiten in Teams (zb mit Branches)
    2. Code ist von überall erreichbar (bei Nutzung von Github/Bitbucket/etc.)
    3. Möglichkeit Code von anderen zu verbessern und zu zeigen ("Pullrequests")
    4. gute Bewerbungsgrundlage, "social Network"-Aspekte
    5. "jeder" nutzt git: Microsoft, Facebook, Amazon, Cisco, T-Mobile, Google, Android ...
